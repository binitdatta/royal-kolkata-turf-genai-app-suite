Spring Boot Spring AI Application Architecture

Spring AI Gen AI Driven Architecture - Operational Use Case

  • 1. There are two main types of users:
  • - Operational Users: Manage daily business tasks by creating and viewing data.
  • - Bot user types can use the same UI based on role-based access.
  • Operational User Flow:
  • 1. Operational User logs in and navigates to the Angular UI.
  • 2. Angular UI invokes backend REST API endpoints based on user actions.
  • 3. Backend REST APIs perform DML operations on the database.
  • 4. The backend REST API returns responses to the Angular UI.
  • 5. Angular UI renders the data, confirms success, or warns about failures.
  • 6. PubSub copies messages published to a Topic to all Subscriptions.
  • 7. Consumers pull messages from Subscriptions independently.
  • 8. If a message is acknowledged by one consumer, it is deleted from that Subscription but remains in others until consumed.

Spring AI Gen AI Driven Architecture - Bot Use Case

  • 1. Now, consider the analytics/managerial user interacting with the bot.
  • 2. The user navigates to the ChatBot, enters a question, and presses the Send button.
  • 3. The Angular ChatBot invokes the backend Gen AI custom REST API (not OpenAI API).
  • 4. The OpenAI custom service sends an initial prompt to invoke specific functions using JSON Schema with Spring AI.
  • 5. The OpenAI LLM API determines which Spring AI function to call based on JSON Schema.
  • 6. A specific Spring AI function invokes a REST API.
  • 7. The REST API retrieves data from the backend SQL database.
  • 8. The backend REST API returns data to the Spring AI function.
  • 9. Angular UI renders the data, confirms success, or warns about failures.
  • 10. The Spring AI function embeds the response and sends it back to OpenAI LLM API.
  • 11. The OpenAI API returns the final response.
  • 12. The Bot REST API sends the final response to the Angular ChatBot.
  • 13. The Angular UI inspects the response and dynamically renders text, tables, or charts.
